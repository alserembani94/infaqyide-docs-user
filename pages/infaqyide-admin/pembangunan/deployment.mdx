# _Deployment_ infaqYIDE Admin

Untuk tujuan deployment, infaqYIDE Admin menggunakan AWS Amplify, dan Github sebagai _CI/CD Automation_. Penggunaan _branch_ amat penting, supaya tidak mengganggu production.

Untuk melihat _configuration_, anda boleh _login_ ke _AWS Console_ dan pergi ke AWS Amplify, dan pilih projek `infaqyide-admin`.

<Bleed>![AWS Amplify Console](https://nimbus-screenshots.s3.amazonaws.com/s/824c58f4d50b4cf6f11cf08d9457c696.png)</Bleed>

Setiap versi yang dilihat dalam AWS Console telah dipautkan dengan _branch_ yang terdapat dalam Github.

Terdapat beberapa perkara yang perlu diketahui berkaitan penggunaan AWS Amplify untuk tujuan _frontend development_:

## 1. Domain

Untuk mengubah _domain_, anda boleh pergi ke menu kiri, dan pilih `Domain Management`. Untuk setiap versi, anda boleh pautkan ke domain yang anda inginkan.

## 2. _Environment Variables_

Ini adalah pengurusan rahsia. Segala rahsia seperti _secret key_ tidak akan (dan tidak sepatutnya) disimpan dalam Github. Oleh itu, sekiranya terdapat _update_, sila tambah di bahagian _environment variables_. Anda juga boleh menetapkan nilai berbeza untuk versi yang berbeza.

---

Untuk menu lain, anda boleh rujuk kepada [dokumentasi AWS Amplify berkaitan _frontend_](https://docs.amplify.aws/).
