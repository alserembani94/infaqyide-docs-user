# Pengurusan Versi infaqYIDE Admin

infaqYIDE Admin menggunakan Github untuk menguruskan versi. Terdapat beberapa _branch_ penting dalam repositori yang perlu diketahui oleh pembangun dan penyelenggara:
- **`main`** - _Branch_ paling stabil, untuk tujuan _versioning_ utama.
- **`production`** - _Branch_ yang digunakan untuk _deploy_ ke _production environment_.
- **`staging`** - _Branch_ yang digunakan untuk menguji _deployement_ untuk tujuan pengujian seperti UAT.

<Callout>Anda boleh membuat _branch_ sendiri dan kemudian membuat _pull request_ dalam Github</Callout>
<Callout type="warning" emoji="⚠️">Sila jangan sentuh _branch_ `production` melainkan anda tahu apa yang anda lakukan.</Callout>

Proses _deployment_ adalah seperti berikut:

## 1. Pengujian

Apabila anda telah _develop_ modul baharu atau perkenalkan _fix_, sila uji di _branch_ `staging` dahulu.

```bash
git checkout staging
git merge <branch-to-merge>
```

## 2. Binaan stabil

Setelah anda pastikan binaan stabil, anda boleh _merge_kan ke main.

```bash
git checkout main
git merge staging
```

## 3. _Production_

Untuk meletakkan _update_ terbaru ke _production_, anda boleh _merge_kan ke _branch_ `production`.

```bash
git checkout production
git merge main
```

## 4. Meletakkan versi

Anda boleh _assign_ versi pada commit yang stabil, menggunakan cara di bawah.

```bash
git tag <versi>
git push origin <versi>
```
